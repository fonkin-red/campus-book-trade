## 一、项目概述  【中期必填】

### 1. 项目背景
随着高校教材更新换代频繁及课外阅读需求增加，学生手中积压了大量闲置二手书籍。传统的毕业季摆摊或QQ群叫卖方式存在信息分散、交易效率低、书籍查找困难等问题。
本项目旨在构建一个专注于校园内部的二手图书交易平台，通过线上信息流管理，盘活校内闲置图书资源，降低学生购书成本，促进绿色阅读循环。

### 2. 项目目标
开发一个基于Web的校园二手图书交易系统，实现以下核心目标：

1.信息透明化：提供图书详情展示（ISBN、新旧程度、价格、卖家信息）。
2.交易流程化：支持买家下单、卖家确认、交易完成的全流程管理。
3.管理便捷化：提供管理员后台，对图书分类、订单状态、用户信息进行维护。
4.过程规范化：通过Git进行严格的版本控制与协作记录。

### 3. 开发环境与条件
开发工具：IntelliJ IDEA (后端), VS Code VUE (前端), Navicat (数据库管理)
数据库：MySQL 8.0.45
运行环境：JDK 21, VUE3.0，spring boot 3.5.14.MyBatis,Element plus,node .js,Maven
版本控制：Git (托管于GitHub)
项目管理：Markdown文档编写，Git Issue/Milestone跟踪进度

## 三、系统设计  【中期必填】

### 1. 系统架构
本系统采用前后端分离的 B/S 架构，分为前端展示层、后端服务层和数据存储层三层。

  **前端展示层**
  - 基于 Vue 3（Composition API + `<script setup>`）构建单页面应用（SPA）
  - 使用 Element Plus 组件库搭建 UI 界面，实现响应式布局
  - 通过 Vue Router 管理前端路由，配合导航守卫实现登录校验和权限控制
  - 使用 Pinia 管理全局状态（用户登录态、Token 等）
  - 通过 Axios 封装 HTTP 请求，统一处理请求头携带 JWT Token 和异常响应

  **后端服务层**
  - 基于 Spring Boot 3 搭建 RESTful API 服务
  - 采用分层架构：Controller（接收请求）→ Service（业务逻辑）→ Mapper（数据访问）
  - 使用 MyBatis 注解方式（@Select、@Insert 等）操作数据库，无需 XML 映射文件
  - 通过 LoginInterceptor 拦截器统一校验 JWT Token，从 Token 中提取用户信息注入请求属性
  - 使用 BCrypt 算法对用户密码进行加密存储
  - 跨域配置（CORS）支持前端开发服务器跨域访问

  **数据存储层**
  - 使用 MySQL 8.0 作为关系型数据库
  - 共设计 7 张数据表：用户表、图书分类表、图书商品表、购物车表、订单表、收藏表、公告表
  - 字符集采用 utf8mb4，支持全文索引（图书名称和作者）

  **认证与交互流程**
  用户（浏览器）→ 登录 → 后端签发 JWT Token → 前端存储 Token
  后续请求 → Axios 拦截器自动附加 Authorization: Bearer  →
  后端 LoginInterceptor 校验 Token → 提取 userId 注入 Request →
  Controller 获取 userId 执行业务逻辑 → 返回 JSON → 前端渲染

代码片段 1：JWT 登录认证（后端）

  位置： UserService.java

  public LoginResponse login(LoginRequest req) {
      // 1. 根据用户名查询用户
      User user = userMapper.selectByUsername(req.getUsername());
      if (user == null || user.getStatus() == 0) {
          throw new RuntimeException("用户不存在或已被禁用");
      }
      // 2. 用 BCrypt 验证密码（不比对原文，防脱库泄露）
      if (!passwordEncoder.matches(req.getPassword(), user.getPassword())) {
          throw new RuntimeException("密码错误");
      }
      // 3. 验证通过，生成 JWT Token（包含 userId, username, role）
      String token = jwtUtil.generateToken(user.getId(), user.getUsername(), user.getRole());
      // 4. 返回 Token + 用户基本信息
      LoginResponse resp = new LoginResponse();
      resp.setToken(token);
      // ... 设置 user 信息
      return resp;
  }

  ▎ 说明：用户登录的核心流程。密码用 BCrypt 加密存储，验证时通过 matches() 比对密文。登录成功后生成 JWT Token
  ▎ 返回前端，后续请求携带 Token 即可识别用户身份，无需在服务端维护 Session。

  代码片段 2：路由守卫（前端）

  位置： router/index.js

  router.beforeEach((to, from, next) => {
    const token = localStorage.getItem('token')
    const user = JSON.parse(localStorage.getItem('user') || '{}')
    // 需要登录但没 Token → 弹提示并跳转到登录页
    if (to.meta.requiresAuth && !token) {
      ElMessage.warning('请先登录')
      next('/login')
    // 需要管理员权限但角色不是 1 → 拒绝
    } else if (to.meta.requiresAdmin && user.role !== 1) {
      ElMessage.warning('权限不足')
      next('/')
    } else {
      next()
    }
  })

  ▎ 说明：前端权限控制的实现。在路由定义中使用 meta 标记哪些页面需要登录（requiresAuth:
  ▎ true）或需要管理员权限（requiresAdmin: true）。每次路由跳转前，beforeEach 导航守卫检查 localStorage 中的 Token
  ▎ 和用户角色，不满足条件则拦截并跳转。

  代码片段 3：图书模糊搜索（后端 MyBatis）

  位置： BookMapper.java

  @Select("SELECT * FROM book WHERE " +
          "(title LIKE CONCAT('%',#{keyword},'%') " +
          "OR author LIKE CONCAT('%',#{keyword},'%')) " +
          "AND status = 1")
  List<Book> searchByKeyword(String keyword);

  ▎ 说明：使用 MyBatis 注解方式直接写 SQL，按书名或作者做 LIKE 模糊查询，同时过滤出状态为 "在售"
  ▎ 的图书。CONCAT('%',#{keyword},'%') 拼接前后百分号实现模糊匹配，相比在 Java 代码里拼好再传入，这种方式能防止 SQL
  ▎ 注入。
