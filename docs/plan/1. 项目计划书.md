## 一、项目概述

### 1. 项目背景
随着高校教材更新换代频繁及课外阅读需求增加，学生手中积压了大量闲置二手书籍。传统的毕业季摆摊或QQ群叫卖方式存在信息分散、交易效率低、书籍查找困难等问题。
本项目旨在构建一个专注于校园内部的二手图书交易平台，通过线上信息流管理，盘活校内闲置图书资源，降低学生购书成本，促进绿色阅读循环。

### 2. 项目目标
开发一个基于Web的校园二手图书交易系统，实现以下核心目标：

1.信息透明化：提供图书详情展示（ISBN、新旧程度、价格、卖家信息）。
2.交易流程化：支持买家下单、卖家确认、交易完成的全流程管理。
3.管理便捷化：提供管理员后台，对图书分类、订单状态、用户信息进行维护。
4.过程规范化：通过Git进行严格的版本控制与协作记录。

### 3. 开发环境与条件
开发工具：IntelliJ IDEA (后端), VS Code VUE (前端), Navicat (数据库管理)
数据库：MySQL 8.0.45
运行环境：JDK 25, VUE
版本控制：Git (托管于GitHub)
项目管理：Markdown文档编写，Git Issue/Milestone跟踪进度

## 二、需求分析

### 1. 目标用户与使用场景
普通学生用户：
场景一：想要卖书 -> 发布二手书信息（上传图片、定价、描述）。
场景二：想要买书 -> 搜索指定教材或小说 -> 查看详情 -> 联系卖家/下单。
场景三：个人中心 -> 管理自己的发布记录和购买订单。

系统管理员：
场景一：审核图书发布内容。
场景二：管理图书分类标签（如：计算机类、文学类、考研资料）。
场景三：处理订单纠纷或维护系统公告。
### 2. 核心功能清单
模块	    功能点	                                
用户模块	注册/登录、个人信息修改、密码找回	      
图书展示	首页推荐、按分类筛选、关键词模糊搜索	
图书发布	上传图片、填写ISBN/书名/售价	
交易管理	加入购物车、生成订单、模拟支付/确认收货	
后台管理	用户管理、图书分类管理、订单数据看板	

### 3. 非功能需求
- 性能要求 页面加载时间 < 5秒。
- 安全要求 密码加密存储。
- 兼容性要求 支持主流浏览器（Chrome, Edge）最新版本。

## 三、技术路线

### 1. 总体架构
采用B/S架构与前后端分离模式。
前端：调用后端接口，渲染页面。
后端：提供JSON数据接口，处理业务逻辑。
数据库：持久化存储用户、图书、订单数据。

### 2. 主要技术
后端框架：Spring Boot 2.x + MyBatis-Plus 
前端框架：Vue 3 + Element Plus UI库
数据库：MySQL 8.0.45
图片存储：本地文件存储映射路径 

### 3. 数据设计初步考虑
- 主要实体或对象 用户(User)、图书(Book)、分类(Category)、订单(Order)、购物车(Cart)。
- 关键数据表或数据关系 个用户对应多本发布的图书 。
一本图书属于一个分类。
一个买家可以生成多个订单，一个订单包含一本或多本图书 (M:N，通过订单详情表实现)。
