# 校园图书交易系统测试样例

## 1. 用户管理模块测试用例

### 1.1 用户注册功能（API测试）
**测试目的**：验证用户注册API是否正常

**前置条件**：系统正常运行，数据库可连接

**测试步骤**：
1. 发送POST请求到 `/user/register`
2. 请求体：`{"username": "testuser", "password": "password123", "confirmPassword": "password123", "email": "test@example.com"}`
3. 验证响应状态码和内容

**预期结果**：
- 响应状态码：200
- 响应内容：`{"code": 0, "msg": "success"}`
- 数据库中新增用户记录
- 用户密码加密存储

**实际结果**：响应状态码200，响应内容为`{"code": 0, "msg": "success"}`，数据库新增用户记录，密码使用BCrypt加密存储。
**测试状态**：✅ 通过

### 1.2 用户登录功能（API测试）
**测试目的**：验证用户登录API是否正常

**前置条件**：已存在有效用户账户

**测试步骤**：
1. 发送POST请求到 `/user/login`
2. 请求体：`{"username": "testuser", "password": "password123"}`
3. 验证响应状态码和内容

**预期结果**：
- 响应状态码：200
- 响应内容包含JWT token
- token有效并可验证

**实际结果**：响应状态码200，响应体包含有效JWT token，token经验证合法且包含用户信息。
**测试状态**：✅ 通过

### 1.3 用户信息获取（API测试）
**测试目的**：验证用户信息获取API是否正常

**前置条件**：用户已登录，有有效token

**测试步骤**：
1. 发送GET请求到 `/user/info`，携带token
2. 验证响应状态码和内容

**预期结果**：
- 响应状态码：200
- 响应内容包含用户信息（密码字段不返回）
- 信息准确无误

**实际结果**：响应状态码200，返回的用户信息包含id、username、email等字段，无密码字段，信息与注册时一致。
**测试状态**：✅ 通过
