# 校园图书交易系统测试计划

## 1\. 测试范围

### 1.1 系统架构

* **前端**：Vue.js + Element UI
* **后端**：Spring Boot + Java
* **数据库**：MySQL
* **技术栈**：JWT认证、RESTful API、文件上传

### 1.2 功能模块

* **用户模块**：注册、登录、个人信息管理
* **图书模块**：图书发布、搜索、浏览、详情查看
* **购物车模块**：添加、修改、删除购物车商品
* **订单模块**：创建订单、支付、发货、收货确认
* **后台管理模块**：

  * 用户管理（启用/禁用）
  * 分类管理（增删改查）
  * 公告管理（发布、编辑、删除）
  * 订单管理（管理员发货）

### 1.3 测试环境

* **前端**：Chrome浏览器最新版本
* **后端**：Spring Boot 2.7.x + Java 11
* **数据库**：MySQL 8.0
* **测试工具**：Postman, JUnit, Mockito

## 2\. 测试目标

* 验证系统各项功能正常运作
* 确保用户流程顺畅
* 检查数据一致性和完整性
* 验证权限控制机制（普通用户/管理员）
* 测试系统性能和稳定性
* 确保API接口符合设计规范

## 3\. 测试策略

### 3.1 功能测试

* 用户认证流程（注册、登录、JWT token）
* 图书管理（发布、搜索、编辑、删除）
* 购物车操作（添加、修改数量、删除）
* 订单全生命周期（创建、支付、发货、收货）
* 后台管理功能（用户状态管理、分类管理、公告管理）

### 3.2 接口测试

* RESTful API功能验证
* 参数校验和错误处理
* 权限控制验证

### 3.3 兼容性测试

* 不同浏览器兼容性（Chrome, Firefox, Edge）
* 不同屏幕尺寸适配

### 3.4 性能测试

* API响应时间
* 并发用户处理能力
* 数据库查询性能

## 4\. 测试资源

* **测试人员**：1人（功能测试、接口测试、性能测试）
* **测试时间**：2026年5月7日 - 2026年5月11日
* **测试工具**：

  * 功能测试：Postman, Chrome开发者工具
  * 单元测试：JUnit 5, Mockito
  * 性能测试：JMeter
  * 代码质量：SonarQube

## 5\. 风险评估

* **高风险**：支付流程、订单状态转换、权限控制
* **中风险**：数据一致性、文件上传、搜索功能
* **低风险**：界面美观性、辅助功能
